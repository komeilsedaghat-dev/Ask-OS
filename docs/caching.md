# SQLite3 Caching Layer

To reduce API usage costs and speed up response times for identical prompts, **Ask-OS** features a local SQLite3-backed caching layer.

## Cache Database Location
The database is located at:
* **Linux/Unix**: `~/.cache/askos/query_cache.db`
* **Windows**: `C:\Users\<Username>\.cache\askos\query_cache.db`

## Database Schema

The database consists of two main tables:

### 1. `command_cache`
Stores generated commands for prompt-model pairs.

| Column | Type | Description |
| :--- | :--- | :--- |
| `prompt` | TEXT | The primary key. Stored as normalized lowercase with stripped whitespaces. |
| `model_name` | TEXT | Part of composite primary key. Matches the model used to query. |
| `command` | TEXT | The raw terminal command generated by the LLM. |
| `created_at` | TIMESTAMP | Creation timestamp. |

### 2. `execution_history`
Tracks execution attempts and exit codes for auditing.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | INTEGER | Auto-incrementing primary key. |
| `prompt` | TEXT | The natural language prompt. |
| `command` | TEXT | The executed command string. |
| `exit_code` | INTEGER | Subprocess return code (`0` for success, `130` for user cancellation). |
| `executed_at` | TIMESTAMP | Execution timestamp. |

## CLI Commands for Cache

### 1. View Cache Stats
Shows cache count and database file size.
```bash
askos cache stats
```

### 2. Clear Cache
Deletes all cached query records.
```bash
askos cache clear
```

### 3. View History
Shows a tabular list of recent executions.
```bash
askos history --limit 20
```
